# 卡牌详情页动态分析报告

**分析日期**: 2026-04-12  
**工具**: mGBA 0.10.5 + mgba-live-mcp  
**ROM**: 游戏王 EX2006（2343.gba）

---

## 一、触发条件

在卡牌列表页面选中一张卡牌，按下 **A 键**，进入卡牌详情页。
此时画面显示：卡牌大图、卡名、星级图标、属性、种族、ATK/DEF 数值、卡牌说明文本。

按键前后共有 **742 处 VRAM 条目发生变化**（BG0区:638, BG1区:104）。

---

## 二、显示模式

| 寄存器 | 值 | 含义 |
|--------|-----|------|
| DISPCNT (`0x04000000`) | `0x1D00` | mode=0，启用 BG2+BG3+OBJ |

> 卡牌列表页使用 BG0+BG1；详情页切换为 BG2+BG3，这是一次完整的显示模式切换。

### BG 层配置

| 层 | BGnCNT 值 | 颜色数 | Tile 数据基址 | 地图基址 | 说明 |
|----|-----------|--------|--------------|---------|------|
| BG2 | `0x1E83` | 256色 | `0x06000000`（char_block=0） | `0x0600F000`（map_block=30） | 卡牌图像 |
| BG3 | `0x1F0B` | 16色 | `0x06008000`（char_block=2） | `0x0600F800`（map_block=31） | 文字/UI 框架 |

## 三、VRAM 布局

### 3.1 BG2 —— 卡牌图像层

- **Tile 数据**: `0x06000000`，256色（64字节/tile），tiles 1–36 有效数据
- **地图数据**: `0x0600F000`，排列为 **3列 × 12行**（即 24px 宽 × 96px 高）
- **说明**: 卡牌艺术图通过 DMA 解压后写入 tiles 1–36；tile 0 为空白填充

```
地图布局（BG2，3×12 tile 网格 = 24×96 像素）:
[0,0] [1,0] [2,0]
[0,1] [1,1] [2,1]
...（共 12 行）
```

### 3.2 BG3 —— 文字/UI 层

- **Tile 数据**: `0x06008000`，16色（32字节/tile），共约 154 个有效 tile
- **地图数据**: `0x0600F800`，30列 × 20行（对应 GBA 240×160 分辨率）

#### BG3 地图行用途

| 行号 | 内容 |
|------|------|
| row 0–1 | **卡牌名称**（tile 16–27 = 12个字符）|
| row 2–4 | UI 分隔线 / 属性图标区 |
| row 5 | ATK / DEF 标签行 |
| row 6–17 | **卡牌说明文本**（tile 100–153）|
| row 18–19 | 按钮区域（"返回"等操作提示）|

#### BG3 Tile 分类

| Tile 编号 | 内容 |
|-----------|------|
| 0 | 空白（全0）|
| 1 | 背景填充 |
| 2–13 | UI 边框（调色板色2填充，`0x22222222...`）|
| 14–15 | UI 边框变体（调色板色E填充，`0xEEEEEEEE...`）|
| **16–27** | **当前卡牌名称字符**（12个字符对应12个tile）|
| 28–99 | UI 图标（属性、种族、ATK/DEF 标签等）|
| **100–153** | **当前卡牌说明文本字符** |

### 3.3 OAM —— 精灵层

| 精灵编号 | Y 坐标 | X 坐标范围 | Tile | 推测用途 |
|----------|--------|------------|------|----------|
| 22–27（6个）| 116 | 16–96（步距16）| 1021 | 星级图标（前半段）|
| 0–6（7个） | 116 | 112–208（步距16）| 1021 | 星级图标（后半段）|
| 9–21（13个）| 92 | 16–208（步距16）| 925 | ATK/DEF 数字或横向装饰条 |
| 7 | 92 | - | - | 纵向边框装饰 |
| 8 | 0 | - | - | 纵向边框装饰 |

> 星级图标共 13 个精灵横排，均使用 tile 1021（16×8 像素，横向精灵，调色板16色）。
> 实际显示的星数通过哪些精灵可见来控制（部分精灵可能通过 X 坐标置屏幕外隐藏）。

## 四、ROM 数据来源

### 4.1 数据压缩

卡牌详情页的 tile 数据（卡图像素 + 字体像素）**均为压缩存储**，无法在 ROM 中直接搜索原始字节。  
游戏在加载时通过 BIOS 解压缩 SWI（LZ77=0x11，Huffman=0x13）将数据写入 VRAM。

### 4.2 已知 ROM 数据位置

| ROM 偏移 | GBA 地址 | 内容 | 编码 |
|---------|---------|------|------|
| `0x114A90` | `0x08114A90` | Huffman 压缩块（magic=0x20）解压约 528KB | 二进制 |
| `0x1DB9C10` | `0x09DB9C10` | 字符串表起始地址 | - |
| `0x4CC4`（相对字符串表）| - | 字符串指针表（4字节/条目）| 小端 32位 |
| `0x1DCC602+` | `0x09DCC602+` | 卡组名字符串 | ASCII |
| `0x1DC9000+` 附近 | `0x09DC9F48+` | 卡牌名/卡组名字符串 | ASCII（卡组名） |
| `0x1DBF000+` 附近 | `0x09DBF03A+` | 卡牌名字符串 | 自定义双字节编码 |

### 4.3 字符编码

- **卡组名**：ASCII 编码（可直接在 ROM hex 中辨认）
- **卡牌名**：自定义编码，字节范围以 `0xF1`–`0xFF` 开头的双字节对
- **说明文本**：同样使用自定义编码

### 4.4 Huffman 块推测

ROM `0x08114A90` 的 528KB Huffman 块可能包含以下之一：
- 完整日文字体 tile 集（16512 tiles × 32字节 = 528KB，与解压大小高度吻合）
- 或大量卡牌艺术图 tile 集

验证方法：解压该块后，搜索 BG3 内存中已知 tile（如 tile 16–27 的 32字节模式）。

## 五、调色板

BG3 使用调色板集 1–3（各16色），BG2 使用全局256色调色板（调色板0）。

| 调色板编号 | 用途 |
|-----------|------|
| pal[0] | BG2 256色（卡图专用，全量调色板）|
| pal[1] | BG3 UI 边框 / 背景色 |
| pal[2] | BG3 文字颜色 |
| pal[3] | BG3 特殊高亮色 |

## 六、分析局限性

| 局限 | 说明 |
|------|------|
| DMA 不可拦截 | `callbacks:add("write",...)` 无法监控 VRAM，只能拍前后快照对比 |
| 无法实时追踪 ROM 读取 | 无法获知解压函数读取了 ROM 的哪个偏移，需通过代码断点或 exec 回调补充 |
| 卡牌 ID 未确认 | 当前截图显示的卡牌 ID 尚未通过 EWRAM 确定 |
| 压缩块未完整解析 | `0x08114A90` Huffman 块的内容尚待进一步反汇编验证 |

## 七、后续工作建议

1. **确定当前卡牌 ID**：扫描 EWRAM（`0x02000000`–`0x02040000`），在卡牌列表页和详情页之间比对差异，找到存储选中卡牌索引的内存地址。

2. **追踪解压调用**：用 `callbacks:add("exec", ...)` 监控 BIOS SWI 0x11/0x13 调用，在解压发生时记录 `r0`（ROM 源地址）和 `r1`（VRAM 目标地址），精确定位卡图数据位置。

3. **建立字符映射表**：读取 BG3 tile 16–27 的像素数据，对照字符串表中的编码值，逐步建立自定义编码到字符的映射。

4. **定位卡牌数据结构**：根据已知字符串表结构，找到卡牌 ID → 名称/描述/属性数值的完整数据结构，用于 ROM hack 修改。
