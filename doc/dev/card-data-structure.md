# 卡牌数据结构调查报告

本文记录对《游戏王 EX2006》（2343.gba）中卡牌相关数据的调查结果，
涵盖卡牌属性数据表、卡名字符串表、卡牌图像格式，以及 LZ77 压缩图形区域。

---

## 一、卡牌属性数据表

### 位置与规模

| 项目 | 值 |
|------|-----|
| ROM 偏移起始 | `0x018169B6` |
| ROM 偏移结束 | `0x01832602` |
| 记录大小 | 22 字节（11 × uint16 LE） |
| 总记录数 | 5170 条 |
| 唯一槽位数 | 2054 个（copy_index=0 的主记录） |
| 槽位 ID 范围 | `0x0000` – `0x19FE` |

### 字段布局

每条记录 22 字节，以 **uint16 小端**格式排列：

| 字节偏移 | 字段名 | 说明 |
|---------|--------|------|
| +00 | `zero_0` | 恒为 `0x0000` |
| +02 | `slot_id` | 卡槽编号（与 `data.md` 中 Slot 列一致） |
| +04 | `copy_index` | 副本索引（0=主记录，1/2/3=第几个副本） |
| +06 | `one` | 恒为 `0x0001`（可能是"有效"标记） |
| +08 | `atk` | 攻击力；Spell/Trap 卡为 `0xFFFF` |
| +10 | `def` | 守备力；Spell/Trap 卡为 `0xFFFF` |
| +12 | `level` | 星数（怪兽卡） |
| +14 | `attribute` | 属性代码（见下表） |
| +16 | `race` | 种族代码（见下表） |
| +18 | `unknown` | 大部分为 0，Spell/Trap 有非零值 |
| +20 | `zero_1` | 恒为 `0x0000` |

### 属性编码（部分确认）

| 值 | 属性 | 验证来源 |
|----|------|---------|
| 1 | LIGHT（光） | Blue-Eyes White Dragon |
| 3 | DARK（暗） | Kuriboh |

> ⚠️ 完整属性/种族编码表尚未全部验证，需对照 `data.md` 进一步比对。

### 种族编码（部分确认）

| 值 | 种族 | 验证来源 |
|----|------|---------|
| 1 | Dragon（龙族） | Blue-Eyes White Dragon |
| 2 | Fiend（恶魔族） | Kuriboh |
| 8 | Spell（魔法卡） | 181 条记录 |
| 9 | Trap（陷阱卡） | 114 条记录 |

### 已验证示例（Blue-Eyes White Dragon）

```
ROM 偏移: 0x018169D4
Raw hex : 0000 A70F 0000 0100 B80B C409 0800 0100 0100 0000 0000

slot_id       = 0x0FA7 (= 4007，与 data.md 吻合)
copy_index    = 0x0000 (主记录)
atk           = 0x0BB8 (= 3000)
def           = 0x09C4 (= 2500)
level         = 0x0008 (= 8星)
attribute     = 0x0001 (LIGHT)
race          = 0x0001 (Dragon)
```

## 二、卡名字符串表

### 位置与格式

| 项目 | 值 |
|------|-----|
| ROM 偏移起始 | `0x015BB5AC` |
| 字符串编码 | CP1252（兼容 Latin-1，含德语 `ä ö ü` 等扩展字符） |
| 排列顺序 | 与卡牌数据表槽位顺序一致 |

每张卡依次排列 **5 个语言**的 null 终止字符串：

| 顺序 | 语言 |
|------|------|
| 1 | EN（英语） |
| 2 | DE（德语） |
| 3 | FR（法语） |
| 4 | IT（意大利语） |
| 5 | ES（西班牙语） |

**对齐规则**：每个字符串（含终止 `\0`）若字节数为奇数，则补一个额外 `\0`，保持 2 字节对齐。

### 已验证示例

```
ROM 0x015BB5AC  → "Blue-Eyes White Dragon\0"     (EN, 24字节，已对齐)
                → "Blauäugiger Weißer Drache\0\0" (DE, 偶数对齐)
                → "Dragon Blanc aux Yeux Bleus\0\0" (FR)
                → "Drago Bianco Occhi Blu\0\0"    (IT)
                → "Dragón Blanco de Ojos Azules\0" (ES)

下一张卡（slot 0x0FA8 = Mystical Elf）紧接其后
```

## 三、卡牌图像数据

### 图像区位置与格式

| 项目 | 值 |
|------|-----|
| ROM 偏移起始 | `0x01000000` |
| ROM 偏移结束 | `0x01463480` |
| 总大小 | ~4.4 MB（= 2054 × 2240 字节） |
| 格式 | **8bpp GBA tiles**（256 色调色板） |
| 每卡尺寸 | 5 tiles × 7 tiles = **40×56 像素** |
| 每卡字节数 | 5 × 7 × 64 = **2240 字节**（stride） |
| 卡牌总数 | **2054** 张（与唯一槽位数一致） |

图像区结束后紧接全零填充，边界精确匹配（`2054 × 2240 = 0x463480`）。

### 8bpp Tile 格式说明

8bpp tiles：每个像素 1 字节，值为调色板索引（0–255）。

每个 8×8 tile 占 **64 字节**，行优先排列：
```
字节 0  → 第0行第0列（色彩索引）
字节 1  → 第0行第1列
...
字节 63 → 第7行第7列
```

tile 按列优先（Column-major）铺排成整张卡：
```
tile 0  = 左上角 (0,0)–(7,7)
tile 1  = (8,0)–(15,7)
...
tile 4  = (32,0)–(39,7)    ← 第一行最右 tile
tile 5  = (0,8)–(7,15)
...
```

### 调色板（未解决）

尚未确认每张卡调色板的存储位置。

已探索区域：

| 区域 | 情况 | 结论 |
|------|------|------|
| `0x00FF0000 – 0x01000000` | 共 128 个 512 字节块 | 所有 pal[0] ≠ 0，不符合 GBA 调色板惯例 |
| `0x01463480` 之后 | 全零至 `0x014634A0` | 非调色板 |
| stride 内嵌（2240+512 = 2752 B/卡） | 未充分验证 | 待测试 |

> **待办**：验证每卡是否内嵌 512 字节调色板（stride = 2752），
> 或调色板存储在其他区域（如 `0x01463480` 之后的低熵区）。

### 图像索引映射

验证表明图像按**顺序索引**排列（不是 slot_id 直接作为索引）：

```
image_index 0 → 卡名表第0张卡
image_index 1 → 卡名表第1张卡
...
地址公式: ROM 偏移 = 0x01000000 + image_index × 2240
```

以 slot_id 直接作索引（Blue-Eyes = 0x0FA7 = 4007）会超出图像区，故排除。

### ⚠️ 当前导出状态（未完成）

调色板尚未确认，目前导出均无法正确着色。

最佳调查成果（`output/cards_8bpp_grid.png`）：
- 灰阶渲染（像素索引值直接作为灰度），可见怪兽轮廓和卡牌边框
- 确认了图像区的位置与格式

关于 `output/card_0000.png` 的发现：
- 该文件显示蓝色背景上的数字字体（0–9 两行）
- **说明图像区起始处存储的是 UI 字体/数字图形，而非卡牌插画**
- 实际卡牌插画从哪个 index 开始尚未确认
- `card_0002.png` 显示水平色条纹，疑为 UI 背景纹理

## 四、LZ77 压缩图形区域

### 主要压缩区（游戏背景画面）

| 项目 | 值 |
|------|-----|
| ROM 偏移范围 | `0x1B8FB8C` – `0x1DBF019` |
| 区域大小 | ~2.3 MB |
| 内容 | 游戏背景画面（推测） |
| 压缩格式 | GBA LZ77（BIOS 解压，header byte `0x10`） |

该区域直接跟在对手图形（`0x1B8FB8B` 结束）之后，在卡组名字符串表（`0x1DBF01A`）之前。

**GBA LZ77 格式说明**：
```
字节 0        = 0x10（LZ77 类型标识）
字节 1–3（LE）= 解压后大小（3字节，最大 16MB）
字节 4+      = 压缩数据（每8位一个 flag byte，0=字面量，1=反向引用）
```

### 标题画面 LZ77

| 项目 | 值 |
|------|-----|
| ROM 偏移 | `0x1EC33DC` |
| 内容 | 标题画面压缩图形 |
| 数据来源 | `doc/um06-romhacking-resource/title-screen.md` |

> 标题画面需专门工具解压处理，已列入 PLAN.md T5。
